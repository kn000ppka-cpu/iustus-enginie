# Каталог норм и правил движка Iustus Engine

**Версия:** 1.0 (рабочая, для v1 продукта)
**Дата:** 25.04.2026
**Статус:** утверждено для имплементации.
**Назначение:** служит единственным источником истины (Single Source of Truth) для движка расчёта. Каждое правило в коде ссылается на запись из этого каталога по `Rule ID`.

## Условные обозначения

- **Rule ID** — стабильный идентификатор правила (используется в коде, тестах, экспорте «Путь машины»).
- **Этап** — этап user flow из PDF (1–6).
- **Источник** — норма УК/УПК/УИК + конкретный пункт ППВС.
- **Статус:** все правила утверждены к имплементации (✅). После релиза v1 каталог уйдёт на верификацию преподавателям РГУП и судьям; их замечания будут вноситься в v1.1.

## Глобальные принципы (применяются ко всем расчётам)

1. **Период УДО не засчитывается как отбытие** наказания. При расчёте погашения судимости, доли отбытия для УДО (ст. 79) и замены (ст. 80) используется только реальный срок отбытия (СИЗО + колония), без периода нахождения на УДО.
2. **Дата приговора = дата вступления приговора в законную силу** (а не дата постановления). Все детекторы Timeline (T-069-5, T-070, T-086) используют именно её.
3. **Изменение категории по ст. 15 ч. 6** — отдельная кнопка в UI «Применить, если возможно». Движок проверяет условия применимости, но не применяет автоматически. После применения — пересчёт всех зависящих правил.
4. **M-070-25** (лимит 35 лет ЛС для террор./экстремистских составов) — включено в v1.
5. **P-072-EXCL** (исключения зачёта 1:1 для особо опасного рецидива и террор. составов) — включено в v1.
6. **Единица расчёта** — день. Все промежуточные значения хранятся в днях, конвертируются в формат `год/месяц/день` только для отображения. Месяц = 30 дней, год = 12 месяцев = 360 дней (для конвертации в эквиваленты ст. 71/72).
7. **Без округлений** — целочисленная арифметика в днях; перевод в годы/месяцы при выводе использует точное деление с остатком.
8. **Только диапазон** — движок выдаёт `[MIN; MAX]`, не предлагает точку.
9. **Последовательность правил** строгая. Изменение порядка ловится юнит-тестом.
10. **Каждый шаг логируется** — для «Пути машины» и аудита.

> Все коэффициенты применяются последовательно к **MAX**, а MIN корректируется только правилами рецидива (ст. 68) и исключительных обстоятельств (ст. 64). Сводка — в схеме `scheme2_coefficients.png`.

| 1 | E-066-3 | УК ст. 66 ч. 3 | Стадия = покушение | `MAX = MAX × 3/4` |

## Формат шаблона приговора (резолютивная часть)

Источник: ППВС № 55 от 29.11.2016, пп. 27–37; УПК ст. 308.

Структура (для каждого эпизода):
```
По ст. {статья ч. {часть}} УК РФ — назначить наказание в виде {вид} сроком {срок}.
```

Совокупность преступлений:
```
На основании ст. 69 ч. {2|3} УК РФ путём {поглощения|частичного|полного} сложения наказаний по совокупности преступлений окончательно назначить {вид} сроком {срок}.
```

Совокупность приговоров:
```
На основании ст. 70 УК РФ к назначенному наказанию частично/полностью присоединить неотбытую часть наказания по приговору {суд} от {дата} и окончательно назначить {вид} сроком {срок}.
```

Зачёт:
```
На основании ст. 72 ч. 3.1 УК РФ зачесть в срок отбытия наказания время содержания под стражей с {дата} по {дата} из расчёта {1:1|1:1.5|1:2}.
```

Вид ИУ:
```
Отбывание наказания назначить в исправительной колонии {общего|строгого|особого} режима / колонии-поселении / воспитательной колонии.
```

Условное:
```
На основании ст. 73 УК РФ назначенное наказание считать условным с испытательным сроком {срок}.
```
